# WebRPA 启动器 - 快速开始

## 📋 前置要求

### 1. 安装 Rust (必需)

WebRPA 启动器使用 Tauri 框架开发,需要 Rust 环境。

**Windows 安装步骤:**

1. 访问 https://www.rust-lang.org/tools/install
2. 下载 `rustup-init.exe`
3. 运行安装程序,选择默认选项
4. 安装完成后,重启命令提示符/PowerShell
5. 验证安装:
   ```bash
   rustc --version
   cargo --version
   ```

**预计安装时间:** 5-10分钟

### 2. 安装 Visual Studio C++ 构建工具 (Windows必需)

Rust 在 Windows 上需要 MSVC 工具链:

1. 访问 https://visualstudio.microsoft.com/zh-hans/downloads/
2. 下载 "Visual Studio 2022 生成工具"
3. 安装时选择 "使用 C++ 的桌面开发"
4. 或者安装完整的 Visual Studio Community

## 🚀 开发模式

### 方式一: 使用批处理脚本 (推荐)

```bash
cd launcher
双击运行 "开发模式.bat"
```

### 方式二: 手动命令

```bash
cd launcher
..\nodejs\npm.cmd run tauri:dev
```

首次运行会下载并编译 Rust 依赖,需要较长时间(10-20分钟)。

## 📦 构建生产版本

### 方式一: 使用批处理脚本 (推荐)

```bash
cd launcher
双击运行 "构建启动器.bat"
```

### 方式二: 手动命令

```bash
cd launcher
..\nodejs\npm.cmd run tauri:build
```

构建完成后,可执行文件位于:
- `launcher/src-tauri/target/release/WebRPA启动器.exe`

安装包位于:
- `launcher/src-tauri/target/release/bundle/msi/` (MSI安装包)
- `launcher/src-tauri/target/release/bundle/nsis/` (NSIS安装包)

## 🎯 使用启动器

1. 将构建好的 `WebRPA启动器.exe` 复制到 WebRPA 项目根目录
2. 双击运行 `WebRPA启动器.exe`
3. 点击"启动服务"按钮
4. 等待服务启动完成(约5-10秒)
5. 自动打开浏览器访问 WebRPA

## ⚠️ 常见问题

### Q1: 启动器无法启动

**A:** 检查以下几点:
- 确保 `Python313` 文件夹存在于项目根目录
- 确保 `nodejs` 文件夹存在于项目根目录
- 确保 `WebRPAConfig.json` 配置文件存在

### Q2: 服务启动失败

**A:** 查看日志区的错误信息:
- 端口被占用: 修改 `WebRPAConfig.json` 中的端口号
- Python 错误: 检查 Python 环境和依赖
- Node.js 错误: 检查 Node.js 环境和依赖

### Q3: 构建失败

**A:** 常见原因:
- Rust 未正确安装: 重新安装 Rust
- MSVC 工具链缺失: 安装 Visual Studio C++ 构建工具
- 网络问题: 使用代理或更换 Rust 镜像源

### Q4: 图标显示异常

**A:** 确保以下图标文件存在:
- `launcher/src-tauri/icons/32x32.png`
- `launcher/src-tauri/icons/128x128.png`
- `launcher/src-tauri/icons/128x128@2x.png`
- `launcher/src-tauri/icons/icon.ico`
- `launcher/src-tauri/icons/icon.icns`

## 🔄 更新版本号

修改以下文件中的版本号:

1. `launcher/package.json` → `"version": "x.x.x"`
2. `launcher/src-tauri/Cargo.toml` → `version = "x.x.x"`
3. `launcher/src-tauri/tauri.conf.json` → `"package": { "version": "x.x.x" }`
4. `launcher/src/App.vue` → `const version = ref('x.x.x')`

## 📝 开发说明

### 项目结构

```
launcher/
├── src/                      # Vue 前端
│   ├── App.vue              # 主组件
│   ├── main.js              # 入口
│   └── style.css            # 样式
├── src-tauri/               # Tauri 后端
│   ├── src/
│   │   └── main.rs          # Rust 主程序
│   ├── icons/               # 应用图标
│   ├── Cargo.toml           # Rust 依赖
│   └── tauri.conf.json      # Tauri 配置
├── package.json             # Node 依赖
├── vite.config.js           # Vite 配置
├── 开发模式.bat             # 开发启动脚本
└── 构建启动器.bat           # 构建脚本
```

### 技术栈

- **前端框架**: Vue 3
- **构建工具**: Vite
- **桌面框架**: Tauri 1.6
- **后端语言**: Rust
- **UI样式**: 原生 CSS

### 修改界面

编辑 `launcher/src/App.vue` 文件:
- `<template>` 部分: HTML 结构
- `<script setup>` 部分: 逻辑代码
- `<style scoped>` 部分: 样式

修改后,开发模式会自动热重载。

### 修改后端逻辑

编辑 `launcher/src-tauri/src/main.rs` 文件。

修改后需要重新编译:
```bash
cd launcher
..\nodejs\npm.cmd run tauri:dev
```

## 🌐 配置 Rust 镜像源 (可选)

如果下载 Rust 依赖很慢,可以配置国内镜像:

创建或编辑 `~/.cargo/config.toml`:

```toml
[source.crates-io]
replace-with = 'ustc'

[source.ustc]
registry = "https://mirrors.ustc.edu.cn/crates.io-index"
```
