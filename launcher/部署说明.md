# WebRPA 启动器部署说明

## 前置要求

### 1. 安装 Rust

访问 https://www.rust-lang.org/tools/install 下载并安装 Rust。

Windows 用户可以直接下载 rustup-init.exe 并运行。

验证安装:
```bash
rustc --version
cargo --version
```

### 2. 安装 Tauri CLI

```bash
cargo install tauri-cli
```

或使用项目内置的 npm:
```bash
cd launcher
..\nodejs\npm.cmd install
```

## 生成图标

### 方法一: 使用生成器 (推荐)

1. 在浏览器中打开 `launcher/生成图标.html`
2. 点击"下载全部"按钮
3. 将下载的 PNG 文件放入 `launcher/src-tauri/icons/` 目录
4. 使用在线工具转换为 ICO 和 ICNS:
   - PNG 转 ICO: https://convertio.co/zh/png-ico/
   - PNG 转 ICNS: https://convertio.co/zh/png-icns/

### 方法二: 使用 Tauri CLI

如果你有自己的图标文件 (建议 1024x1024 PNG):

```bash
cd launcher
npm run tauri icon path/to/your/icon.png
```

这会自动生成所有需要的图标文件。

## 构建应用

### 开发模式

```bash
cd launcher
开发模式.bat
```

或手动运行:
```bash
cd launcher
..\nodejs\npm.cmd run tauri:dev
```

### 生产构建

```bash
cd launcher
构建启动器.bat
```

或手动运行:
```bash
cd launcher
..\nodejs\npm.cmd run tauri:build
```

构建完成后，可执行文件位于:
- Windows: `launcher/src-tauri/target/release/WebRPA启动器.exe`
- 安装包: `launcher/src-tauri/target/release/bundle/`

## 部署步骤

1. 确保已安装 Rust 和 Tauri CLI
2. 生成并放置图标文件
3. 运行构建脚本
4. 将生成的 `WebRPA启动器.exe` 复制到 WebRPA 项目根目录
5. 用户双击 `WebRPA启动器.exe` 即可启动

## 常见问题

### Q: 构建失败，提示找不到 Rust

A: 确保已正确安装 Rust，并重启终端/命令提示符。

### Q: 图标显示不正确

A: 确保所有图标文件都已正确放置在 `src-tauri/icons/` 目录，并且格式正确。

### Q: 启动器无法找到 Python 或 Node.js

A: 确保 `Python313` 和 `nodejs` 文件夹在 WebRPA 项目根目录。

### Q: 服务启动失败

A: 检查日志区的错误信息，确保端口未被占用，配置文件正确。

## 更新版本

修改以下文件中的版本号:

1. `launcher/package.json` - "version" 字段
2. `launcher/src-tauri/Cargo.toml` - "version" 字段
3. `launcher/src-tauri/tauri.conf.json` - "package.version" 字段
4. `launcher/src/App.vue` - version ref 的默认值

然后重新构建即可。

## 自动更新功能

启动器内置了版本检查功能，可以通过修改 `src-tauri/src/main.rs` 中的 `check_update` 函数来实现:

1. 连接到你的更新服务器或 GitHub Releases
2. 比较版本号
3. 返回更新信息和下载链接

示例:
```rust
#[tauri::command]
async fn check_update(current_version: String) -> Result<VersionInfo, String> {
    let response = reqwest::get("https://api.github.com/repos/your-repo/releases/latest")
        .await
        .map_err(|e| e.to_string())?;
    
    let release: GitHubRelease = response.json()
        .await
        .map_err(|e| e.to_string())?;
    
    Ok(VersionInfo {
        current_version,
        latest_version: release.tag_name,
        has_update: release.tag_name != current_version,
        update_url: release.html_url,
    })
}
```

## 技术栈

- **前端**: Vue 3 + Vite
- **后端**: Rust + Tauri
- **UI**: 原生 CSS (无第三方UI库)
- **进程管理**: Rust std::process
- **日志**: Tauri Event System
